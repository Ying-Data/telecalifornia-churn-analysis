Set Correct Column Data Types
This step prevents chart errors and ensures calculations work correctly

Why this matters

Power BI auto-detects types, but sometimes gets it wrong. If "Age" is treated as Text instead of Number, you can't create an average. If "Churn_Probability" is Text, you can't create a gauge. Fix these now — it takes 2 minutes and saves hours of frustration.

In Power Query Editor — set each column type

1
Click on the column header "Age" → in the ribbon, click "Data Type" dropdown → select "Whole Number"
2
Click column "Monthly Charge" → Data Type → "Decimal Number"
3
Click column "Total Revenue" → Data Type → "Decimal Number"
4
Click column "Churn_Probability" → Data Type → "Decimal Number"
5
Click column "Latitude" → Data Type → "Decimal Number". Do same for "Longitude"
6
Click column "Zip Code" → Data Type → "Text" (ZIP codes should never be treated as numbers)
7
When prompted "Change column type?" → click "Replace current"
8
Click "Close & Apply" button (top-left of Power Query Editor ribbon)
9
Wait for the loading spinner to complete. You'll return to the main Power BI canvas.
✅ Checkpoint — Right-panel "Fields"
  • You see "telecom_clean_powerbi" table in the right-side Fields panel
  • Expand it — you see all 57 columns listed
  • Numeric columns show a Σ (sigma) icon next to them
  • Text columns show an ABC icon
  • Latitude/Longitude show a globe icon

Create DAX Measures
These are the calculated KPIs that power your KPI cards and visuals

How to create a measure (do this for EVERY measure below)

1
In the top ribbon, click the tab "Modeling"
2
Click "New Measure" button (has an fx icon)
3
A formula bar appears at the top with "Measure = " pre-filled
4
Delete "Measure =" and paste in the DAX formula from below
5
Press Enter to confirm. The measure appears in the Fields panel.
6
Right-click the measure in Fields panel → "Rename" if you want a cleaner name
DAX MEASURE 1 — Total Churn Rate
Churn Rate = 
DIVIDE(
    CALCULATE(COUNTROWS('telecom_clean_powerbi'), 'telecom_clean_powerbi'[Customer Status] = "Churned"),
    CALCULATE(COUNTROWS('telecom_clean_powerbi'), 'telecom_clean_powerbi'[Customer Status] <> "Joined"),
    0
)
DAX MEASURE 2 — Total Revenue Lost
Revenue Lost = 
CALCULATE(
    SUM('telecom_clean_powerbi'[Total Revenue]),
    'telecom_clean_powerbi'[Customer Status] = "Churned"
)
DAX MEASURE 3 — Churned Customer Count
Churned Customers = 
CALCULATE(
    COUNTROWS('telecom_clean_powerbi'),
    'telecom_clean_powerbi'[Customer Status] = "Churned"
)
DAX MEASURE 4 — Avg Monthly Charge (Churners)
Avg Charge Churners = 
CALCULATE(
    AVERAGE('telecom_clean_powerbi'[Monthly Charge]),
    'telecom_clean_powerbi'[Customer Status] = "Churned"
)
DAX MEASURE 5 — Critical Risk Count
Critical Risk Count = 
CALCULATE(
    COUNTROWS('telecom_clean_powerbi'),
    'telecom_clean_powerbi'[Churn_Risk_Tier] = "Critical Risk"
)
DAX MEASURE 6 — Estimated Annual Revenue at Risk
Annual Revenue at Risk = 
CALCULATE(
    SUMX(
        FILTER('telecom_clean_powerbi', 'telecom_clean_powerbi'[Churn_Risk_Tier] = "Critical Risk"),
        'telecom_clean_powerbi'[Monthly Charge] * 12
    )
)
DAX MEASURE 7 — Net Retention Rate
Retention Rate = 
1 - [Churn Rate]

Simulator Tab — "What If" Churn Reduction
The most impressive tab — let your boss play with retention scenarios

Create a What-If Parameter

1
Add new page → rename: "Retention Simulator"
2
In the top ribbon → click tab "Modeling"
3
Click "New Parameter" button → select "Numeric range"
4
Name: "Churn Reduction %"
5
Minimum: 0 | Maximum: 15 | Increment: 1 | Default: 5
6
Check "Add slicer to page" → click Create
7
A slider slicer appears on the page. Move it and watch the measures update!
DAX — Simulated Revenue Saved
Simulated Revenue Saved = 
VAR ChurnReduction = 'Churn Reduction %'[Churn Reduction % Value] / 100
VAR BaseChurners = [Churned Customers]
VAR RetainedCustomers = ROUND(BaseChurners * ChurnReduction, 0)
VAR AvgMonthlyCharge = [Avg Charge Churners]
RETURN
    RetainedCustomers * AvgMonthlyCharge * 12
DAX — Simulated Customers Retained
Simulated Retained = 
ROUND([Churned Customers] * ('Churn Reduction %'[Churn Reduction % Value] / 100), 0)
DAX — Net ROI (after program cost)
Simulated Net ROI = 
[Simulated Revenue Saved] - 150000
